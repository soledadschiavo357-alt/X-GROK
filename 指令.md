# Google Search Console 结构化数据问题修复报告

## 1. 问题描述
根据 Google Search Console 的报告，网站首页 (`index.html`) 的结构化数据 (Structured Data) 存在以下三个警告/错误：

1.  **未填写字段 "deliveryTime"** (在 "offers.shippingDetails" 中)
2.  **字段 "unitCode" 中的枚举值无效** (在 "offers.shippingDetails.deliveryTime.transitTime" 中)
3.  **字段 "unitCode" 中的枚举值无效** (在 "offers.shippingDetails.deliveryTime.handlingTime" 中)

这些问题主要影响 Google 商家列表 (Merchant Listings) 的富媒体搜索结果展示。

## 2. 问题分析

### 2.1 缺少 `deliveryTime`
在部分 `Offer` 对象（如 "Grok 4.1 现有账号代充" 和 "Grok 4.1 入门共享号"）中，虽然定义了 `shippingDetails`，但缺少了关键的 `deliveryTime` 字段。Google 要求必须明确发货和运输时间。

### 2.2 `unitCode` 枚举值无效
原代码中使用了 `"unitCode": "HUR"` (Hour)。虽然这在 UN/CEFACT 标准中是有效的，但 Google 的结构化数据规范通常推荐使用 `"DAY"` (Day) 作为发货和运输时间的计量单位。

## 3. 修复方案

已对 `index.html` 中的 JSON-LD 数据块进行了以下修改：

### 3.1 修正 `unitCode`
将所有 `"unitCode": "HUR"` 更改为 `"unitCode": "DAY"`，以符合 Google 推荐标准。

### 3.2 补全 `deliveryTime`
为所有缺失该字段的 Offer 补充了 `deliveryTime` 对象。考虑到是虚拟/数字商品，均设置为 0-1 天或即时交付 (0 天)。

**修改后的代码示例：**

```json
"deliveryTime": {
  "@type": "ShippingDeliveryTime",
  "handlingTime": {
    "@type": "QuantitativeValue",
    "minValue": 0,
    "maxValue": 0,
    "unitCode": "DAY"
  },
  "transitTime": {
    "@type": "QuantitativeValue",
    "minValue": 0,
    "maxValue": 0,
    "unitCode": "DAY"
  }
}
```

## 4. 验证与后续
1.  **验证**：建议使用 [Google 富媒体搜索结果测试工具](https://search.google.com/test/rich-results) 或 [Schema Markup Validator](https://validator.schema.org/) 对首页 URL 进行重新验证。
2.  **生效时间**：提交修正后，Google Search Console 通常需要几天时间重新抓取并更新状态。
